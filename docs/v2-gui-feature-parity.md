# v2 PyQt GUI Feature Parity Audit

이 문서는 React/Vite prototype의 디자인 철학을 PyQt main GUI v2로 이식하면서 v1 기능을 잃지 않았는지 추적하는 감사표다. 기계 검증의 원본은 `tests/migration/feature_matrix.json`이고, 수동 Windows 검증 원본은 `docs/migration-smoke-checklist.md`다.

## 판정 기준

- v2 main GUI는 `src/gui/v2/main_window.py`의 `V2MainWindow`가 `src/gui/main_window.py`의 `MainWindow`를 상속해 기존 CRUD, Beholder, tray, sidebar, dashboard, session, notification 경계를 재사용한다.
- prototype 시각 철학은 card/topbar/message banner, compact spacing, light/dark token, status color indicator로 이식하되, OS 기본 제목 표시줄을 유지하고 과한 gradient를 쓰지 않는다.
- 새 설정 UX는 `src/gui/v2/settings_dialog.py`의 `SettingsDialogV2`가 기존 mature 설정 dialogs를 하나의 tab hub로 감싸며, 저장은 기존 DataManager/Beholder actor boundary를 통과한다.
- 아래 “잔여 smoke”가 있는 항목은 자동 검증이 통과해도 Windows runtime smoke 전에는 최종 parity 완료로 선언하지 않는다.

## 현재 검증 상태

- 자동 검증 gate: `python tools/verify_project.py --full`이 pytest, real AppData clone checks, compileall, frontend/dashboard build, cargo check, Tauri release build, migration audit, v2 completion audit trace를 실행한다.
- 이 문서의 parity 근거는 자동 테스트와 코드 상속/라우팅 구조의 증거다.
- Windows-only smoke는 아직 사람이 실행해야 하는 stop condition이다. smoke가 완료되기 전에는 “v1/v2 feature parity 1:1 완전 완료”로 표시하지 않는다.
