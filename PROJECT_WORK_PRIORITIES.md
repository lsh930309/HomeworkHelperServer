# HomeworkHelper 작업 목표 우선순위

이 문서는 2026-05-22 기준 아이디어 박스를 실제 작업 순서로 고정한 상위 목표 목록이다. 세부 설계와 구현 범위는 각 우선순위 착수 시 별도 계획으로 확정한다.

## 0. 코드베이스 정리 및 빌드 자동화 개선

- 루트 문서는 `README.md`와 이 우선순위 문서를 제외하고 `docs/` 아래로 정리한다.
- `build.version.json`을 추적되는 버전 source of truth로 삼고, 호스트/클라이언트 버전 및 build 번호를 빌드 성공 시 갱신한다.
- OS별 자동 타깃 선택, GUI 중심 버전 확인, archive pruning, 선택적 release 게시를 `build.py` 흐름에 통합한다.
- 민감 복구 파일은 `local-artifacts/secrets/` 아래의 Git 미추적 영역에 둔다.

## 1. Tailscale/SSH/자동 실행 안정화

- 호스트와 클라이언트의 Tailscale 의존성, 연결 자동 복구, 자동 실행 UX를 안정화한다.
- SSH 원격 진단/실기기 테스트 워크플로우를 반복 가능한 운영 도구로 확장한다.

## 2. 신호 송수신 및 스마트 폴링 개선

- 호스트/클라이언트 간 상태 신호 송수신을 더 빠르고 견고하게 만든다.
- 비동기 처리, 상태 변화 기반 갱신, 스마트 폴링 레이트를 검토하여 체감 반응성을 개선한다.

## 3. Moonlight/Sunshine 연동 spike

- macOS 클라이언트만으로 원격 게임 환경에 빠르게 전환할 수 있도록 Moonlight client 바인딩을 검토한다.
- Sunshine/스트리밍 실행 상태와 HomeworkHelper Remote UX를 자연스럽게 연결하는 최소 확장 지점을 찾는다.

## 4. 호스트 GUI 마이그레이션

- 단기적으로는 기존 PySide6 마이그레이션 계획을 우선 검토/정리한다.
- Flutter 전환은 macOS 클라이언트 수준의 유려한 UI/UX를 호스트에도 전파할 수 있는지 장기 기술 검토 항목으로 분리한다.
